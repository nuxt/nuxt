---
title: '<ServerOnly>'
description: Render content only on the server and preserve it as static HTML without client hydration.
---

The `<ServerOnly>` component renders slot content only on the server. On the client, the server-rendered HTML is preserved as static content and never hydrated—slot logic (e.g. `new Date()`) does not run again after hydration. The rendered content is still included in the HTML sent to the browser and visible in the page source.

Use for content that must be fixed at render time: build timestamps, server-generated public content, or any markup that should not change after hydration.

::warning
Do not use `<ServerOnly>` for secrets or PII, since rendered content is visible in the page HTML.
::

::note
Content inside `<ServerOnly>` is only preserved when the page already contains server-rendered markup (for example, the initial SSR or prerendered response). On client-side navigation there is no server DOM for `elToStaticVNode` to reuse (e.g., SPA navigation to a non-SSR route), so the implementation falls back to an empty wrapper rather than rendering the slot content.
::

## Example

```vue [app/pages/example.vue]
<template>
  <footer>
    <ServerOnly>
      <!-- Rendered once at build/request time, never updated on client -->
      Generated at {{ new Date().toISOString() }}
    </ServerOnly>
  </footer>
</template>
```

## Use Cases

- **Build timestamps**: Show when the page was generated without the date updating after hydration
- **Server-generated public content**: Display values computed on the server that should not re-run on the client (visible in page source)
- **Static server markup**: Any content that must remain exactly as rendered on the server

## Comparison with Server Components

Unlike [server components](/docs/4.x/directory-structure/app/components#server-components), `<ServerOnly>` accepts arbitrary slot content without creating a separate component. Use it when you need a quick wrapper for server-only markup, similar to how `<ClientOnly>` wraps client-only content.
