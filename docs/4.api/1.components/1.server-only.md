The `<ServerOnly>` component renders slot content only on the server. On the client, the server-rendered HTML is preserved as static content and never hydrated—slot logic (e.g. `new Date()`) does not run again after hydration.

Use for content that must be fixed at render time: build timestamps, server-only data, or any markup that should not change after hydration.

::note
Content inside `<ServerOnly>` is not shown when navigating client-side to a page that was not server-rendered (e.g. SPA navigation to a route without SSR). For pre-rendered or SSR pages, the content is always visible.
::

## Example

```vue [app/pages/example.vue]
<template>
  <footer>
    <ServerOnly>
      <!-- Rendered once at build/request time, never updated on client -->
      Generated at {{ new Date().toISOString() }}
    </ServerOnly>
  </footer>
</template>
```

## Use Cases

- **Build timestamps**: Show when the page was generated without the date updating after hydration
- **Server-only data**: Display values computed on the server that should not re-run on the client
- **Static server markup**: Any content that must remain exactly as rendered on the server

## Comparison with Server Components

Unlike [server components](/docs/4.x/directory-structure/app/components#server-components), `<ServerOnly>` accepts arbitrary slot content without creating a separate component. Use it when you need a quick wrapper for server-only markup, similar to how `<ClientOnly>` wraps client-only content.
