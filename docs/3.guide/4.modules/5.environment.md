description: "Learn how you configure your module to use imports from other modules, typescript decorators or other important things."
---

Here are some common patterns used by module authors.

## Create a nuxt.config.js

To configure your development environment please add a new file named "nuxt.conifg.js".

```js
export default {
    ...
}
```

## Use other modules for development processes

If you want to use components, composables, utilities, and other elements from other Nuxt modules or other libraries, you need to add these modules to the "Modules" section of the Nuxt configuration.

```js
export default {
    modules: [
        '@pinia/nuxt',
    ]
}
```

Than you can use in this example imports from @pinia/nuxt.

## Use typescript decorators

If you want to define and use your own typescript decorators, you need to add these to the typescript section of the Nuxt configuration.

```js
export default {
    tsConfig: {
      compilerOptions: {
        emitDecoratorMetadata: true,
        experimentalDecorators: true,
      },
    }
}
```

Now you can define and use custom typescript decorators.
