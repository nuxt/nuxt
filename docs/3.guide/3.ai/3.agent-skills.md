## What are Agent Skills?

Agent skills are specialized instructions and documentation that help AI coding assistants (like Claude Code, Cursor, and Windsurf) understand how to use your module effectively. When users install a module with `nuxt module add`, the CLI can automatically install the associated skills to their AI tools.

Skills are served via the [`.well-known/skills/`](https://github.com/anthropics/anthropic-cookbook/tree/main/misc/prompt_caching) protocol using [agent-skills-server](https://github.com/anthropics/anthropic-cookbook/tree/main/misc/agent-skills-server).

## For Module Authors

Module authors can distribute skills alongside their modules by:

1. Setting up `agent-skills-server` in their documentation site
2. Creating skill files in the `skills/` directory
3. Declaring the skills URL in the module meta

### Step 1: Install agent-skills-server

In your documentation project (usually a separate `docs/` folder), install the skills server:

```bash [Terminal]
pnpm add agent-skills-server
```

### Step 2: Configure the Nuxt Module

Add the module to your docs `nuxt.config.ts`:

```ts [docs/nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['agent-skills-server/nuxt'],
  skills: {
    dir: './skills', // default
  },
})
```

### Step 3: Create Your Skill

Create a skill folder with a `SKILL.md` file:

```
docs/
├─ skills/
│  └─ my-module/
│     ├─ SKILL.md          # Required
│     └─ references/       # Optional extra docs
├─ nuxt.config.ts
└─ ...
```

The `SKILL.md` file uses YAML frontmatter followed by markdown content:

```md [skills/my-module/SKILL.md]
---
name: my-module
description: Use when working with @nuxtjs/my-module for feature X
license: MIT
metadata:
  version: "1.0.0"
---

# My Module

Instructions for AI assistants on how to use this module.

## When to Use

- Installing and configuring @nuxtjs/my-module
- Using the module's composables
- Customizing module behavior

## Key APIs

### `useMyFeature()`

Returns reactive state for feature X.

## Examples

```vue
<script setup>
const { data } = useMyFeature()
</script>
```
```

#### Frontmatter Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | 1-64 chars, lowercase + hyphens, must match folder name |
| `description` | Yes | 1-1024 chars, explains when to use this skill |
| `license` | No | SPDX identifier (e.g., `MIT`, `Apache-2.0`) |
| `compatibility` | No | Version constraints (max 500 chars) |
| `allowed-tools` | No | Space-separated list of tools the skill can use |
| `metadata` | No | Key-value pairs for additional info |

### Step 4: Declare Skills in Module Meta

In your module definition, add the `agents.skills` configuration:

```ts [src/module.ts]
import { defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: '@nuxtjs/my-module',
    configKey: 'myModule',
    agents: {
      skills: {
        url: 'https://my-module.nuxt.dev',
        skills: ['my-module'],
      },
    },
  },
  setup(options, nuxt) {
    // module setup...
  },
})
```

The `agents.skills` configuration accepts:

| Field | Type | Description |
|-------|------|-------------|
| `url` | `string` | Base URL serving `/.well-known/skills/` |
| `skills` | `string[]` | Skill names to install (optional - if omitted, all skills are offered) |

### Step 5: Verify Your Setup

Start your docs server and verify the endpoint works:

```bash [Terminal]
# Start docs server
pnpm dev

# Check endpoint
curl http://localhost:3000/.well-known/skills/
# → { "skills": [{ "name": "my-module", "description": "...", "files": [...] }] }

# Test with skills CLI
npx skills add http://localhost:3000 --list
```

## For Users

When you run `nuxt module add` for a module that provides skills, you'll be prompted to install them:

```bash [Terminal]
$ nuxt module add @nuxt/ui

✓ Installed @nuxt/ui
✓ Added @nuxt/ui to modules

◐ Checking for agent skills...
◇ Found 1 skill(s)

? Install agent skill(s): nuxt-ui? (Y/n)

◐ Installing nuxt-ui...
✓ Installed to detected agents

✓ Done
```

The skills CLI automatically detects which AI tools you have configured (Claude Code, Cursor, etc.) and installs the skills to the appropriate locations.

### Manual Installation

You can also install skills manually using the skills CLI:

```bash [Terminal]
# Install skills from a URL
npx skills add https://ui.nuxt.com

# List available skills first
npx skills add https://ui.nuxt.com --list

# Install specific skills
npx skills add https://ui.nuxt.com --skill nuxt-ui
```

## Example: Nuxt UI

Nuxt UI provides skills that help AI assistants understand its component library:

```ts [packages/ui/src/module.ts]
export default defineNuxtModule({
  meta: {
    name: '@nuxt/ui',
    configKey: 'ui',
    agents: {
      skills: {
        url: 'https://ui.nuxt.com',
        skills: ['nuxt-ui'],
      },
    },
  },
})
```

When users install Nuxt UI, the CLI offers to install the `nuxt-ui` skill which teaches AI assistants about:

- Available components (Button, Modal, Form, Table, etc.)
- Component props and slots
- Theming with Tailwind Variants
- Best practices and common patterns
