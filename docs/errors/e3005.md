# E3005

The `execute` function from `useAsyncData()` was passed directly as a `watch` callback. When used this way, `watch` passes the new value as the first argument to `execute`, which changes its behavior.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Writing `watch(someRef, execute)` which passes the new ref value as the first argument to `execute`
- Accidentally using the destructured `execute` as a direct callback

## Resolution

Wrap in an inline function to avoid passing arguments:

```ts
watch(someRef, () => execute())
```
