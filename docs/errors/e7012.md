# E7012

The handler passed to `useAsyncData` is not a function. `useAsyncData` expects a function that returns a promise with the data to fetch.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Passing a non-function value as the handler to `useAsyncData`
- Accidentally passing the result of a fetch call instead of a function that performs the fetch

## Resolution

Pass a function as the handler to `useAsyncData`:

```ts
// Incorrect
useAsyncData('key', $fetch('/api/data'))

// Correct
useAsyncData('key', () => $fetch('/api/data'))
```
