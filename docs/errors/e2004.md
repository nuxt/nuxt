# E2004

A route middleware name was referenced that does not exist. This typically happens when a middleware is specified in `definePageMeta()` but there is no corresponding middleware file or registration.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Typo in the middleware name specified in `definePageMeta()`
- The middleware file was deleted or renamed without updating references

## Resolution

Check that the middleware name matches a file in the `middleware/` directory. Middleware names are derived from the filename (e.g., `middleware/auth.ts` becomes `auth`).
