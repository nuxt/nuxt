# E1008

A `vue:setup` hook callback returned a Promise instead of being synchronous. The `vue:setup` hook must be synchronous to ensure proper component initialization.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- The `vue:setup` hook callback returns a Promise instead of being synchronous
- Using `async` keyword on the `vue:setup` callback function

## Resolution

Ensure `vue:setup` callbacks are synchronous. If you need to perform async operations, use a different hook or handle the async logic separately.
