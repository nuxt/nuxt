# B5006

You are using hash-based filenames in your webpack configuration while in development mode. Hash-based filenames are intended for production builds and can cause issues during development, including problems with hot module replacement (HMR).

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- Using hash-based filenames in the dev webpack config
- A production-only webpack option was applied globally instead of conditionally

## Resolution

Remove the hash option from your development configuration, or make it conditional:

```ts
export default defineNuxtConfig({
  webpack: {
    filenames: {
      // Only use hashes in production
    },
  },
})
```
