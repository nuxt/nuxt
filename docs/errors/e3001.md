# E3001

`useFetch()` was called with a URL that starts with `//` (a protocol-relative URL). Nuxt does not support protocol-relative URLs in `useFetch()` because they can cause inconsistent behavior between server and client.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Using protocol-relative URLs like `//api.example.com/data`
- Accidentally doubling the leading slash in a relative path

## Resolution

Use a full URL with an explicit protocol (e.g., `https://api.example.com`) or a relative path (e.g., `/api/data`). For external APIs, set `baseURL` in the `useFetch()` options.
