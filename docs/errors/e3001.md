# E3001

`useFetch()` was called with a protocol-relative URL (starting with `//`). These are not supported because they resolve differently between server and client, causing inconsistent behavior.

## Resolution

Use a full URL with an explicit protocol (e.g. `https://api.example.com`) or a relative path (e.g. `/api/data`). For external APIs, set `baseURL` in the `useFetch()` options. Check for an accidentally doubled leading slash in relative paths.
