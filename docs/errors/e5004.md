# E5004

You attempted to set response headers (e.g., via `useResponseHeader()`) on the client side. Response headers can only be set during server-side rendering.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Calling `useResponseHeader()` or similar composables on the client side
- Missing server-side guard around header-setting logic

## Resolution

Guard the call with `import.meta.server` to ensure it only runs during SSR:

```ts
if (import.meta.server) {
  useResponseHeader('X-Custom-Header', 'value')
}
```
