# E7006

Multiple composables or components are setting the same cookie, causing one value to override another. This can lead to unexpected behavior.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Multiple components calling `useCookie()` with the same key and setting different values
- Duplicate cookie-setting logic across different parts of the application

## Resolution

Ensure only one composable instance sets each cookie. If multiple components need access to the same cookie, use a shared composable or state management to coordinate writes.
