# B6003

The `unhead.legacy` Nuxt config option is deprecated and will be removed in a future release. It exists to keep the Unhead v2 compatibility shims (`hid`, `vmid`, `children`, `body: true`, and unresolved promise values) working when upgrading to Unhead v3, but those patterns are no longer supported going forward.

::note
**This is a build warning (logged).** This warning is logged during the build process. The build will continue, but the issue should be investigated.
::

## Common causes

- `nuxt.config` still sets `unhead.legacy: true` after upgrading to Nuxt 4 / Unhead v3
- `useHead()` / `useSeoMeta()` calls in the codebase still use `hid`, `vmid`, `children`, `body: true`, or pass promise values directly

## Resolution

Remove deprecated head patterns and migrate promise values to resolved values before passing to `useHead`, then drop the `unhead.legacy` option from `nuxt.config`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  // remove this:
  // unhead: { legacy: true },
})
```

Replace `hid` / `vmid` with the `key` property, drop `body: true`, and `await` any promise-returning values before handing them to `useHead`:

```ts
const title = await fetchTitle()
useHead({ title })
```
