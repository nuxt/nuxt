# E3004

Two `useAsyncData()` / `useFetch()` calls share the same key but pass incompatible options. When a cache key is shared, the `handler`, `transform`, `pick`, `getCachedData`, `default`, and `deep` options must all be consistent. This often happens when auto-generated keys collide.

## Resolution

Give each call a distinct key, or extract the shared fetching logic into a composable that both components reuse so the options stay identical.
