# E3004

Two components are using `useAsyncData()` (or `useFetch()`) with the same key but incompatible options. When multiple components share a cache key, their handler, transform, pick, `getCachedData`, default, and deep options must be consistent.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Two components using the same key with different `handler`, `transform`, `pick`, `getCachedData`, `default`, or `deep` options
- Auto-generated keys colliding between different `useFetch()` calls

## Resolution

Use different keys for each `useAsyncData()`/`useFetch()` call, or extract the shared data fetching logic into a shared composable that both components use.
