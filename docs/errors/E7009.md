# E7009

The key passed to `useState` is not a string. `useState` requires a string key to identify the state across server and client.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Passing a non-string value (e.g., number, object, symbol) as the key
- Using a variable that is not of type string

## Resolution

Pass a string key to `useState`:

```ts
useState('myKey', () => initialValue)
```
