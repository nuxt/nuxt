# B3012

A call to `addComponent()` is overriding an existing component without specifying a priority. This can lead to unpredictable component resolution order.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- addComponent() overrides existing component without specifying priority
- Multiple modules registering the same component name

## Resolution

Set the `priority` option when calling `addComponent()` to explicitly control which component takes precedence.
