# E3006

The `useAsyncData()` handler function returned `undefined` on the server. This usually indicates a missing `return` statement in the handler. Returning `undefined` can cause hydration issues because the data will not be serialized and transferred to the client.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- No `return` statement in the `useAsyncData()` handler function
- Early return without a value in some code paths

## Resolution

Always return a value from the handler. Return `null` instead of `undefined` if no data is available.
