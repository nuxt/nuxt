# E3003

`useFetch()` or `useAsyncData()` was called after the component has already been mounted. These composables are designed for use during setup and SSR. After mount, use `$fetch()` directly instead.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Calling `useFetch()` or `useAsyncData()` inside an event handler, `watch` callback, or `setTimeout`
- Using these composables in code that runs after the component's initial setup phase

## Resolution

Use `$fetch()` for data fetching after the component has mounted. Reserve `useFetch()` and `useAsyncData()` for the setup phase only.
