# E1004

`setInterval` was called during server-side rendering. Intervals created during SSR will never be cleared and can cause memory leaks and unexpected behavior.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Calling `setInterval` in `setup()` without guarding for client-side execution
- Third-party libraries using `setInterval` during SSR

## Resolution

Guard with `import.meta.client`, or move the call to `onMounted()` / `onNuxtReady()` to ensure it only runs on the client.
