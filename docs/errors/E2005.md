# E2005

`useRoute()` was called inside a route middleware. Within middleware, the route has not yet been resolved, so `useRoute()` may return unexpected results. Middleware receives `to` and `from` route objects as parameters instead.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Using `useRoute()` inside middleware instead of the `to` and `from` arguments
- Calling a composable that internally uses `useRoute()` from within middleware

## Resolution

Use the `to` and `from` parameters passed to the middleware function instead of `useRoute()`:

```ts
export default defineNuxtRouteMiddleware((to, from) => {
  // Use `to` instead of useRoute()
})
```
