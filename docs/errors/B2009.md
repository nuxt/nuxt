# B2009

A circular dependency was detected in the plugin dependency graph. For example, plugin A depends on plugin B, which in turn depends on plugin A.

::note
**This is a build error (logged).** This error is logged during the build process. The build may continue, but the issue should be investigated.
::

## Common causes

- Plugin A depends on B which depends on A
- Indirect circular chains through multiple plugins

## Resolution

Break the circular dependency by refactoring the plugin logic. Extract shared functionality into a separate composable or utility that both plugins can use independently.
