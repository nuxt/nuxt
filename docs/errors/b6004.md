# B6004

The `experimental.headNext` Nuxt config option is deprecated. CAPO (Critical, Asynchronous, Pre-fetch, Other) head tag sorting is now the default behaviour in Unhead v3 and no longer needs to be opted into.

::note
**This is a build warning (logged).** This warning is logged during the build process. The build will continue, but the issue should be investigated.
::

## Common causes

- `nuxt.config` still sets `experimental.headNext: false` to opt out of CAPO sorting

## Resolution

Remove `experimental.headNext` from `nuxt.config`. CAPO sorting is now the default:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  // remove this:
  // experimental: { headNext: false },
})
```

If you need to opt out temporarily while migrating, set `unhead.legacy: true` instead. Note that `unhead.legacy` is itself deprecated (see `NUXT_B6003`) and will be removed in a future release.
