# B7003

The server bundle must produce a single entry, but `optimization.splitChunks` is enabled for the server webpack config. Chunk splitting is unsupported server-side and fails the build — commonly caused by a global webpack config applied to both client and server.

## Resolution

Disable `splitChunks` for the server build (apply splitting to the client only if needed):

```ts
export default defineNuxtConfig({
  webpack: {
    optimization: {
      splitChunks: false, // server must stay single-entry
    },
  },
})
```
