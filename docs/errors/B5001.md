# B5001

The `compatibilityDate` property is not set in your Nuxt configuration. This date is used by Nuxt to determine which behavior defaults to apply, ensuring your project remains stable across Nuxt updates.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- No `compatibilityDate` set in `nuxt.config`
- The property was accidentally removed during config refactoring

## Resolution

Add `compatibilityDate` to your `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  compatibilityDate: '2024-01-01',
})
```
