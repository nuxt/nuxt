# B4014

A call to `addLayout()` attempted to register a layout with a name that already exists. The existing layout will not be replaced unless explicitly overridden.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- addLayout() for layout that already exists
- Multiple modules registering layouts with the same name

## Resolution

Set `override: true` to replace the existing layout, or use a different name:

```ts
addLayout({ src: resolve('./runtime/layouts/custom.vue'), filename: 'custom.vue' }, 'custom', { override: true })
```
