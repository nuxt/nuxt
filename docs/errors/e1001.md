# E1001

A composable that requires access to the Nuxt instance was called outside of a plugin, Nuxt hook, Nuxt middleware, or Vue setup function. This typically happens when composables like `useNuxtApp()`, `useRoute()`, or `useFetch()` are called in an async callback where the setup context has been lost.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Calling `useNuxtApp()`, `useRoute()`, `useFetch()`, or any composable inside `setTimeout`, `.then()`, or other async callbacks where the setup context is lost
- Using composables in plain utility functions not called from `setup()`, a plugin, or middleware

## Resolution

Capture composable results inside `setup()` / plugin / middleware, then use the captured references later. For server-side async contexts, use `nuxtApp.runWithContext()`.
