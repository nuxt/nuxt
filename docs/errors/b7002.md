# B7002

The Vite `optimizeDeps` configuration is out of date. This can result in slower development server startup or unexpected behavior with pre-bundled dependencies.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- Vite `optimizeDeps` configuration is out of date after dependency changes
- Cached dependency optimization data is stale

## Resolution

Update your `optimizeDeps` configuration, or clear the Vite cache:

```bash
# Clear Vite cache
rm -rf node_modules/.vite
```

Then restart your development server.
