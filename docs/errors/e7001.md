# E7001

A full URL with a hostname was used where a relative payload URL is expected. Payload URLs must be relative paths without a protocol or hostname.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Full URL with hostname used for a payload request (e.g., `https://example.com/api/data`)
- Accidentally passing an absolute URL instead of a relative path

## Resolution

Use relative paths for payload URLs (e.g., `/api/data` instead of `https://example.com/api/data`).
