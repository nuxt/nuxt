# E7007

The `init` parameter passed to `useState` is not a function. `useState` expects a function that returns the initial value, not the value itself.

::note
**This is a runtime error.** Your application will throw when this condition is encountered in both development and production.
::

## Common causes

- Passing a raw value instead of a function as the `init` parameter
- Accidentally omitting the arrow function wrapper

## Resolution

Wrap the initial value in a function:

```ts
// Incorrect
useState('key', myValue)

// Correct
useState('key', () => myValue)
```
