# B4004

Two page files generate the same route name. Route names must be unique for proper navigation and route resolution.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- Two pages generate same route name
- Pages in different directories that resolve to the same route

## Resolution

Set a custom name using `definePageMeta` in one of the conflicting pages:

```ts
definePageMeta({
  name: 'my-custom-route-name'
})
```
