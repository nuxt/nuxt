# E5005

The app manifest fetched from `builds/meta/<buildId>.json` was not a valid manifest object. This usually indicates that the hosting or proxy layer in front of Nuxt is rewriting the manifest URL to an HTML fallback (for example a single-page-application 200 rewrite) instead of returning JSON.

::note
**This is a runtime error (thrown).** This error is thrown at runtime and will halt the manifest fetch.
::

## Common causes

- Hosting platform rewrites all unknown paths to `index.html`, so `builds/meta/*.json` returns HTML
- A reverse proxy strips the `.json` extension or rewrites the response body
- The build was not deployed correctly and the meta files are missing

## Resolution

Ensure `builds/meta/*.json` is served as JSON by your hosting or proxy and is not rewritten to an HTML fallback. Most static hosts let you exclude `builds/meta/*` from SPA fallback rules:

```text
# Example: Netlify _redirects — exclude builds/meta from the SPA fallback
/builds/meta/*  200
/*              /index.html  200
```

If you're behind a CDN or proxy, check that the `Content-Type` of the manifest URL is `application/json`.
