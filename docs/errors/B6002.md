# B6002

A custom composable name conflicts with a Nuxt built-in auto-import. When two auto-imports share the same name, Nuxt cannot determine which one to use, leading to unexpected behavior.

::note
**This is a build error (logged).** This error is logged during the build process. The build may continue, but the issue should be investigated.
::

## Common causes

- A custom composable has the same name as a Nuxt built-in composable
- A third-party module registers an auto-import that conflicts with a Nuxt built-in

## Resolution

Rename your custom composable to avoid the conflict:

```ts
// Instead of useHead.ts (conflicts with Nuxt built-in)
// Rename to useCustomHead.ts
export function useCustomHead() {
  // ...
}
```
