# E2007

`setPageLayout()` was called in a component's `setup()` function during server-side rendering. On the server, layout changes must be done through middleware or `definePageMeta()` to ensure correct SSR output.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Calling `setPageLayout()` in a component's `setup()` function during SSR
- Attempting to dynamically change layout on the server from within a component

## Resolution

Call `setPageLayout()` from middleware instead, or use `definePageMeta({ layout: '...' })` for static layout assignment.
