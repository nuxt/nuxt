# E2008

`setPageLayout()` was called during client-side hydration, which can cause hydration mismatches because the layout was already determined during server-side rendering.

::note
**This is a development warning.** This warning only appears in development mode and does not affect production builds.
::

## Common causes

- Calling `setPageLayout()` in `onMounted()` during the initial hydration phase
- Attempting to change layout during the first client-side render

## Resolution

Use `definePageMeta({ layout: '...' })` for static layout assignment, or call `setPageLayout()` from middleware where it runs before rendering.
