# B4013

A call to `addRouteMiddleware()` attempted to register a middleware with a name that already exists. The existing middleware will not be replaced unless explicitly overridden.

::note
**This is a build warning.** The build will continue, but you should address this issue.
::

## Common causes

- addRouteMiddleware() with name that already exists
- Multiple modules registering middleware with the same name

## Resolution

Set `override: true` to replace the existing middleware, or use a different name:

```ts
addRouteMiddleware({ name: 'auth', path: resolve('./runtime/middleware'), override: true })
```
